Issue Priority URLs % Total Description How to Fix
Response Codes: Internal Client Error (4xx)
 High 3 0.07% Internal broken links returning 4xx errors (typically 404 Page Not Found). These waste crawl budget and create dead-ends for users and search engines. Identify the 3 broken URLs using the Screaming Frog inlinks tab. Update internal links to correct destinations or implement 301 redirects.
Images: Missing Alt Text
 Low 303 28.64% Images have an alt attribute present but the attribute value is empty. This is an accessibility failure and prevents search engines from understanding image content. Add descriptive, keyword-relevant alt text to all 303 images. Prioritise images on key landing pages. Decorative images should use alt="".
Images: Missing Alt Attribute
 Low 266 25.14% Images are missing the alt attribute entirely — not just empty. This violates WCAG accessibility standards and is a missed SEO signal opportunity. Add alt attributes with descriptive text to all 266 images. This is a developer task — bulk-export the list from Screaming Frog and pass to the dev team.
URL: Contains Space
 Low 12 0.31% 12 URLs contain literal spaces, which are unsafe characters. These can break link sharing, cause incorrect encoding, and result in 404 errors in some browsers. Rename these URLs replacing spaces with hyphens. Implement 301 redirects from the old spaced URLs to the corrected versions.
Pagination: Pagination URL Not in Anchor Tag
 High 28 1.20% 28 paginated URLs referenced in rel="next"/"prev" attributes are not linked via standard HTML anchor tags. This means Google cannot crawl between paginated pages and PageRank cannot flow. Ensure all paginated URLs are linked to via standard <a href=""> tags on the page. This allows search engines to crawl and pass PageRank across the paginated series.

Warnings to Address Soon

These 23 warnings are important issues that won't immediately break the site but will hurt crawlability, indexation, security, and rankings if left unresolved. Each unresolved warning deducts 1 point from the health score.

Issue Priority URLs % Total Description How to Fix
Canonicals: Canonicalised
 High 1,943 83.07% 83% of all pages have a canonical pointing to a different URL. While canonicals are legitimate, this high ratio may indicate internal linking to non-canonical URLs throughout the site. Audit internal links and update them to point directly to canonical URLs rather than their non-canonical equivalents. This improves crawl efficiency.
Directives: Noindex
 High 46 1.18% 46 pages have a noindex directive, instructing search engines not to index them. These need to be reviewed — any important pages accidentally noindexed will be invisible in search results. Review all 46 noindex pages. Remove the directive from any pages that should be indexed. Keep it only for truly non-indexable pages (e.g. thank-you pages, internal tools).
Response Codes: Blocked by Robots.txt
 High 4 0.10% 4 internal URLs are blocked by robots.txt. If these pages contain content Google should index, this is a critical indexation failure. Review the robots.txt file. If these URLs should be crawlable, update the disallow rules. Also remove internal links to pages that should remain blocked.
Directives: Nofollow
 High 8 0.21% 8 pages have a meta nofollow directive, preventing PageRank from flowing to linked pages. This is usually applied by mistake alongside noindex. Remove nofollow from pages that should pass PageRank. Only keep it where absolutely intended (e.g. login pages, admin areas).
Pagination: Non-Indexable
 High 21 0.90% 21 paginated URLs are non-indexable (noindex or canonicalised to page 1). Google recommends all pages in a paginated series be indexable and accessible. Ensure paginated pages are indexable and have self-referencing canonicals — not canonicals pointing to the first page in the series.
